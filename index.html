SolarCharge

Smart Solar-Powered Free Charging Stations Database System

System Title

SolarCharge: Smart Solar-Powered Free Charging Stations

A comprehensive database system designed to manage smart solar-powered charging stations that provide both free and premium charging services. The system handles user management, subscription plans, energy consumption tracking, and station monitoring through an integrated web-based platform.

Rationale

Problem Statement

Current public charging infrastructure often lacks the flexibility and efficiency needed to serve diverse user needs. Most systems do not effectively balance free and premium access, leading to limited availability, poor resource allocation, and user dissatisfaction. Additionally, these systems frequently lack real-time monitoring and management capabilities, hindering administrators from making informed decisions.

There is often an overlap between user and administrator functionalities, which can compromise both usability and privacy. Furthermore, many systems are not designed to scale, and suffer from poor data integrity and lack of support for analytics or historical tracking, making optimization difficult over time.

Solution Approach

The SolarCharge database system is designed to support a comprehensive solar-powered charging network that ensures efficiency, flexibility, and security. It implements a dual-tier service model to handle both free and premium access, along with detailed usage tracking and energy monitoring for optimal resource use.

The system clearly separates user roles from administrator capabilities, providing real-time system visibility at appropriate levels while safeguarding user privacy. Its scalable, normalized database structure allows for seamless expansion and maintains data integrity through consistent relationships. Additionally, the inclusion of historical data tracking supports advanced analytics and system optimization to continuously improve performance and service delivery.

Objectives

Primary Objectives

  • Support User Subscription Management:
    • Track subscription plans, purchases, and energy usage limits.
    • Enable tiered access to premium features based on subscription level.
    • Monitor daily energy consumption against quota limits.
  • Enable Station and Port Monitoring:
    • Provide real-time visibility of port occupancy status.
    • Monitor energy generation, consumption, and battery status.
  • Facilitate Administrative Oversight:
    • Allow administrators to see which specific users are using charging stations.
    • Monitor technical performance metrics of stations and ports.
    • Support maintenance tracking and scheduling.
  • Enable User Station Status Visibility:
    • Allow users to view current availability of charging stations.
    • Provide appropriate station status information based on user role.
  • Support Transaction Processing:
    • Track payments, subscription activations, and renewals.
    • Maintain financial records for system sustainability.
  • Record Session Data:
    • Capture detailed information about each charging session.
    • Link sessions to users, ports, stations, and subscriptions.
    • Calculate energy consumption for quota management.
  • Facilitate System Communication:
    • Support notifications to users about charging status.
    • Enable technical alerts for system maintenance.

Database Architecture

Architecture Overview

Components

"SolarCharge: Smart Solar-Powered Free Charging Stations" is a web-based platform designed to provide free, sustainable smartphone charging through solar-powered stations, supported by a robust three-tier DBMS architecture. The system is structured based on an ERD (Entity-Relationship Diagram) that maps the relationships between users, subscriptions, charging sessions, charging ports, and station management. In the Presentation Tier, users access a mobile-friendly web interface to locate stations, start charging, view usage history, and receive notifications. Admins use a secure dashboard to manage stations, monitor performance, and schedule maintenance. The Application Tier acts as the system’s brain, processing business logic such as user authentication, subscription validation, session tracking, and solar energy allocation based on predefined usage plans. It also ensures fair access by enforcing time or energy limits and handling notifications. Finally, the Database Tier securely stores all system data, including user profiles, session logs, real-time charging status, energy usage, and maintenance records. By combining solar technology with intelligent data management, SolarCharge offers a scalable, eco-friendly, and user-friendly solution for public smartphone charging needs—especially in outdoor, remote, or high-traffic urban environments.

SQL Queries

Sample Queries for System Objectives

Query 1: User Subscription Management

        SELECT
            u.user_id,
            u.username,
            sp.plan_name,
            us.start_date,
            us.end_date,
            sp.data_usage_limit,
            COALESCE(SUM(d.total_energy_used_mah), 0) AS total_energy_used,
            sp.data_usage_limit - COALESCE(SUM(d.total_energy_used_mah), 0) AS remaining_mah
        FROM
            user u
        JOIN
            user_subscription us ON u.user_id = us.user_id
        JOIN
            subscription_plan sp ON us.plan_id = sp.plan_id
        LEFT JOIN
            daily_energy_usage d ON u.user_id = d.user_id AND d.usage_date BETWEEN us.start_date AND us.end_date
        WHERE
            us.is_active = TRUE AND u.is_active = TRUE
        GROUP BY
            u.user_id, sp.plan_name, us.start_date, us.end_date, sp.data_usage_limit;

Query 2: Enable Station and Port Monitoring

        SELECT
            rs.station_id,
            cs.station_name,
            rs.status_time,
            rs.battery_level_percentage,
            rs.solar_input_watts,
            rs.power_consumption_watts,
            rs.active_free_ports,
            rs.total_free_ports,
            rs.active_premium_ports,
            rs.total_premium_ports,
            rs.system_operational
        FROM
            real_time_station_status rs
        JOIN
            charging_station cs ON rs.station_id = cs.station_id
        WHERE
            rs.status_time = (
                SELECT MAX(status_time) FROM real_time_station_status WHERE station_id = rs.station_id
            );

Query 3: Facilitate Administrative Oversight

        SELECT
            a.admin_id,
            a.full_name AS admin_name,
            cs.station_id,
            cs.station_name,
            cs.location_description,
            cs.is_active
        FROM
            admin a
        LEFT JOIN
            charging_station cs ON a.admin_id = cs.assigned_admin_id
        ORDER BY
            a.admin_id;

Query 4: Enable User Station Status Visibility

        SELECT
            psv.station_id,
            cs.station_name,
            psv.last_updated,
            psv.available_free_ports,
            psv.available_premium_ports,
            psv.estimated_wait_time,
            psv.is_operational,
            psv.status_message
        FROM
            public_station_view psv
        JOIN
            charging_station cs ON psv.station_id = cs.station_id
        WHERE
            cs.is_active = TRUE;

Query 5: Support Transaction Processing

        SELECT
            p.payment_id,
            u.user_id,
            u.username,
            p.subscription_id,
            p.amount,
            p.payment_method,
            p.payment_date,
            p.payment_status
        FROM
            payment p
        JOIN
            user u ON p.user_id = u.user_id
        ORDER BY
            p.payment_date DESC
        LIMIT 10;

Query 6: Record Session Data

        SELECT
            cs.session_id,
            u.user_id,
            u.username,
            cs.port_id,
            cs.energy_consumed_mah,
            cs.start_time,
            cs.status,
            cs.is_premium
        FROM
            charging_session cs
        JOIN
            user u ON cs.user_id = u.user_id
        WHERE
            cs.status = 'Active';

Query 7: Facilitate System Communication

        SELECT
            sm.station_id,
            cs.station_name,
            sm.maintenance_date,
            sm.maintenance_type,
            sm.maintenance_description,
            sm.performed_by,
            sm.next_schedule
        FROM
            station_maintenance sm
        JOIN
            charging_station cs ON sm.station_id = cs.station_id
        ORDER BY
            sm.maintenance_date DESC;
